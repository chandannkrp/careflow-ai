Emergency triage MVP

CareFlow AI Project Synopsis

CareFlow AI is a hospital operations tool for capturing patient intake, producing deterministic urgency assessments, and keeping the treatment queue ordered by priority, escalation state, score, and wait time.

The MVP is deliberately focused: staff create intakes, the backend scores and inserts patients into the queue, triage staff update status or override priority, and operations teams monitor queue load and wait-time trends.

Application Lifecycle

The core lifecycle begins when staff capture intake details and ends when queue state feeds treatment, metrics, and audit views.

End-to-end patient flow
1ArrivalPatient enters emergency intake by walk-in, ambulance, transfer, or referral.
2IntakeStaff record identifiers, age band, complaint, symptoms, vitals, risk flags, and notes.
3AssessmentBackend applies deterministic scoring from risk flags, vitals, pain, age, and arrival mode.
4Queue EntryThe patient is inserted or refreshed in the treatment queue with category, score, wait start, and status.
5ReviewStaff filter, sort, update status, and escalate or override when authorized.
6OperationsMetrics summarize volume, critical/high load, wait times, and override count.

API Surface

The API is intentionally small and workflow-oriented. The frontend proxies /api to the local backend during Vite development.

Local backend: localhost:8080
Endpoint Purpose Primary Consumer Status
POST /api/intakes Create patient intake, run scoring, and insert or refresh queue entry. IntakeForm via createIntake() Implemented
GET /api/intakes/{id} Retrieve intake details and the latest assessment. Patient detail workflow Backend
POST /api/intakes/{id}/assess Regenerate deterministic urgency assessment and update queue state. Reassessment workflow Backend
GET /api/queue Return sorted queue with optional category, department, and status filters. QueueTable via getQueueEntries() Implemented
POST /api/queue/{patientId}/status Update a patient's queue status after role permission check. QueueTable via updateQueueStatus() Implemented
POST /api/queue/{patientId}/override Apply staff priority override and persist override history. Priority override workflow Backend
GET /api/metrics/queue Return total volume, critical/high count, wait-time metrics, and override count. Metrics dashboard workflow Backend

Implementation Status

The implemented slice already supports the core demo path. Remaining MVP views are planned around patient review, override UX, metrics polish, AI assistance, and hardening.

As documented in project plan

Implemented Now

Intake form, queue table, deterministic scoring, queue filters, local sort controls, status updates, queue metrics service, and demo seed schema support.

Partially Ready

Backend endpoints exist for intake retrieval, reassessment, priority override, and queue metrics, but the corresponding frontend experiences still need completion.

Future Phase

OpenAI-assisted symptom structuring, urgency signal summaries, confidence, missing-detail detection, robust audit views, and role-management polish.

Safety posture AI output is advisory only. Backend rules and staff review remain the source of truth for final category, score, queue ordering, status, and override history.
